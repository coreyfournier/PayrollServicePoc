The Problem

EWA is great — but employees have questions

No transparency

Employees see a number but don't know how it was calculated

Confusing limits

"Why can I only withdraw $175?" — they don't understand the rules

Support overhead

Balance questions become support tickets that humans have to answer

Static interfaces

Traditional apps show a number with no explanation or context

What if an AI assistant could explain this conversationally — using the same payroll data your system already has?

Under the Hood

What Claude can do

ToolPurpose
get_all_employeesFind by name

get_tax_informationWithholding
get_deductionsBenefits
get_ewa_balanceBalance + limit

Automatic chaining

Claude chains tools: name lookup → ID → balance. Up to 10 round-trips per conversation.

Read-only by design

"Change my tax withholding" → politely declines. No write operations available.

Context-aware

Conversation history maintained across turns. Follow-ups work naturally.

Key Takeaways

Why this matters

1

AI + payroll data = self-service support

Employees get instant answers instead of filing tickets. Reduces support load, improves experience.

2

Tool calling is the pattern

Claude doesn't hallucinate numbers — it calls real APIs and calculates from real data. Trustworthy results.

3

We built this in a hackathon

Chat UI, balance engine, withdrawal rules, full Docker deployment — all wired into the existing event-driven architecture.
